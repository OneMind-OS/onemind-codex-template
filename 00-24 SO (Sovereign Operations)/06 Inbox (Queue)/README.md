# 06 — Inbox (Queue)

> Everything comes here first. Nothing dies here.

---

## What Lives Here

This is the **capture bucket** for the entire vault. When something comes in and you're not sure where it goes — or you don't have time to route it properly — drop it here. The Inbox is the entry point of the CODEX Capture step.

**Rule:** Nothing stays in Inbox forever. Process it within 48 hours or during your weekly review.

---

## How to Use Inbox

### Capturing

Drop any of the following here without overthinking:
- Voice memo transcriptions
- Screenshots of ideas
- URLs you want to save
- Quick notes from meetings
- AI outputs you want to keep
- Tasks that don't have a home yet

### Processing

During your daily or weekly review, go through Inbox and:
1. For each item — ask: *What is this? What should I do with it?*
2. Route it to the right domain folder
3. Apply the right template if it needs structure
4. Delete it if it's no longer relevant

---

## Zeus's Inbox Discipline

I process my inbox every evening as part of my daily review. My rule: **the inbox is never more than 20 items**. When it hits 20, I stop capturing and process first.

My AI agent (Legacy) helps me process inbox during our morning check-in. I describe items verbally, Legacy suggests routing, and I confirm.

---

## Processing Checklist

When processing each item, decide:

- [ ] Is this actionable? → Add to task manager (Todoist) AND file the context note
- [ ] Is this a decision? → Use `decision.md` template, save to relevant domain
- [ ] Is this a reference? → File directly to the correct domain
- [ ] Is this a project? → Use `project.md` template
- [ ] Is this still relevant? → If not, delete it

---

## Note

You'll notice the Inbox folder is in `00-24 SO` — domain `06`. This is intentional. The Inbox belongs to your Sovereign Operations layer because processing the inbox is a *system operation* — it's how information enters and routes through the entire vault.
