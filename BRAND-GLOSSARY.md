## CODEX (The Method)

| | |
|---|---|
| **What it is** | The 5-step method for processing everything in your life: **C**apture → **O**rganize → **D**irect → **E**xecute → **×** Multiply |
| **Full meaning** | C.O.D.E.× — an acronym where each letter is a step, and × (multiply) represents the compounding review loop |
| **What it's NOT** | OpenAI Codex (that's a coding agent). Not just a filing system. Not just an acronym. |
| **The dual meaning** | "Codex" means "ancient book of knowledge" (Latin). The acronym C.O.D.E.× spells CODEX. Method AND metaphor align perfectly. |
| **When to use** | Teaching the method, explaining the process, framework discussions |
| **Examples** | "Run it through the CODEX." / "The CODEX method has 5 steps." / "C.O.D.E.× — that's how you process everything." |

### The 5 Steps

| Step | Name | What Happens |
|------|------|-------------|
| **C** | Capture | Grab anything — thought, file, link, idea. Zero friction. |
| **O** | Organize | Route it to the right domain. Add metadata. |
| **D** | Direct | Prioritize. Assign operators (human or AI). Define success. |
| **E** | Execute | Do the work. Agents and humans act on directed items. |
| **×** | Multiply | Review, measure, extract lessons. Feed insights back. Compound. |

## Common Confusions

### "Is CODEX the method or the folder?"
**Both.** Just like PARA is both Tiago Forte's method (Projects, Areas, Resources, Archives) AND the folder structure you create. CODEX is the method (C.O.D.E.×) AND the vault (your files). Context makes it clear.

### "Is One Mind the same as One Mind OS?"
**No.** One Mind is the philosophy/brand. One Mind OS is the complete system (method + vault + tools + community). One Mind is the WHY. One Mind OS is the WHAT.

### "Is this the same as OpenAI Codex?"
**No.** OpenAI Codex is a coding agent for software engineers. One Mind Codex is a life operating system for everyone. Different audiences, different purpose, different product. Always say "One Mind Codex" or "The CODEX Method" to disambiguate.

### "What's the difference between SO and the life domains?"
**SO is the engine. HP/LE/GE are the vehicle.** SO is where your agents, tools, automations, and system infrastructure live. The three life domains — they hold the actual content of your life. SO runs them.

### "Where does code go?"
- **System code** (scripts, configs, deployments) → `_system/` folder (excluded from public template)
- **Framework code** (skills, templates, conventions) → `_codex/` folder
- **Domain documentation about tech** → SO domains (02 Agents, 08 Infrastructure, 10 Development)
- **Business tech stack** → GE/79 Technology

### "What's _codex vs _system?"
- **`_codex/`** = The intelligence layer. Templates, skills, conventions, profiles. Ships with the public template.
- **`_system/`** = The operations layer. Scripts, API keys, deployment configs. Private — never shared.
